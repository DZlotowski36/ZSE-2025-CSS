Budowa selektora

  1. 1. Wymień oraz opisz dwie części selektora

    Selektor to część reguły CSS, która wskazuje, do którego elementu HTML odnosi się stylizacja. Część ta może być różnorodna – może odnosić się do klasy, identyfikatora lub tagu HTML.

    Deklaracja zawiera zestaw właściwości i ich wartości, które mają zostać zastosowane do wybranego elementu.

  2. 2. Znaczenie nawiasów klamrowych {...}

    Nawiasy klamrowe {...} są używane do otaczania deklaracji CSS w selektorze. Wewnątrz nawiasów umieszczamy właściwości, które chcemy zastosować do wybranego elementu HTML. Przykładowo:

    p {
    color: red;
    font-size: 16px;
    }

    W tym przykładzie nawiasy klamrowe otaczają deklaracje stylów dla elementu p.

  3. 3. Opisz cechy

    Cechy CSS to właściwości, które mogą być przypisane do elementu. Każda cecha ma określoną listę możliwych wartości, które mogą być użyte. Na przykład cecha text-align dotyczy wyrównania tekstu i może przyjąć wartości takie jak: left, right, center, justify. Cechy są kluczowe do określania wyglądu elementów w dokumencie.

  4. 4. Opisz wartości

    Wartości to konkretne ustawienia przypisane do cech. Każda cecha ma predefiniowane wartości, które mogą zostać zastosowane w zależności od potrzeby. Na przykład, dla cechy text-align, wartości mogą obejmować: left, right, center, justify. Przypisanie niewłaściwej wartości (np. przypisanie koloru do text-align) jest błędem.

  5. 5. Kiedy stosujemy znak średnika (;), kiedy nie jest konieczny

    Średnik (;) w CSS jest używany do oddzielania poszczególnych właściwości w deklaracji. Jeśli mamy więcej niż jedną właściwość w deklaracji, każdą musimy zakończyć średnikiem, np.:

    p {
    color: red;
    font-size: 16px;
    }

    Średnik jest niekonieczny po ostatniej właściwości w deklaracji, ale dla zachowania porządku i zgodności składniowej często jest używany po każdej właściwości.
